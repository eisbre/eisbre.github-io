递归递推

##引例

  • 递归：从已知问题的结果出发，用迭代表达式逐步推算出问题的开始的条件，即顺推法的逆过程，称为递归。
  • 递推：递推算法是一种用若干步可重复运算来描述复杂问题的方法。递推是序列计算中的一种常用算法。通常是通过计算机前面的一些项来得出序列中的指定象的值。
  • 递归与递推区别：相对于递归算法,递推算法免除了数据进出栈的过程，也就是说,不需要函数不断的向边界值靠拢,而直接从边界出发,直到求出函数值。

在算法题中，递归需要注意复杂度，当递归层数过深时可能会出现爆桟，超时的问题。关于递归改非递归，有一套固定方案。

理解递归可以利用用递归判定树或者递归搜索树

经典例子就是斐波那锲数列

详细内容以及矩阵运算 + 快速幂优化算法看这里传送门这里仅提到递归和递推两种算法

递归
递归计算的节点个数是 O(2n)O(2n) 的级别的，存在大量重复计算。
时间复杂度是 O(2n)O(2n)，一秒内大约能算到第三四十项。

以5为例，树节点代表数列对应下标的值

其中f[0]=0,f[1]=1。父节点为子节点的和。

先由上层分解，直到能确定值为止再由叶子结点求和回到父节点，父节点就是最终结果。如果使用数组进行记忆化，可剪掉部分分支，只要计算过一次，就会有保留的值，在程序确定值后，就可进行回溯。

递推

递推。
开一个大数组，记录每个数的值。用循环递推计算。
总共计算 n 个状态，所以时间复杂度是 O(n)。
但需要开一个长度是 n 的数组，内存将成为瓶颈，当 n=108 时，需要的内存是 4∗1081024×1024≈381MB4∗1081024×1024≈381MB。
分子中乘4是因为C++中 int 类型占4字节。

递推和递归相反，是从部分问题逐步求解再到整体。递归是将整体分解成部分问题，再进行求解。这里的部分问题具有相似性。递推，递归都有规律可寻。

递归实现指数型枚举

递归在算法中重要的应用就是DFS。这里我们用树的方式来展现枚举的过程。

以3为例，要从其中随机选取任意多个，输出所有可能的选择方案，我们可以用一个数组（程序中为dis）。当数组值为1时，表示选择了这个数，对应的数即为下标，于是将这个数输出；当数组值为2时，代表没用选这个数，则不输出；当数组值为0时，代表此位没有确定是否选择。**要注意这里的未确定和未选择是不一样的。当dis[i]=2时，表示i这个数不选，是确定的状态，而等于零则代表还没确定选还是不选。**通过观察上面的判定树，我们可以发现，叶子结点即为结果，在到达叶子时，即可遍历数组，将值为1的对应下标输出。此时数组应都是2或者1，不存在0。当然这里的回溯将状态改回可以省略，这样写只是更加易懂，说明DFS在回溯，进行另一个分支时的状态应该是之前的原状态。

###递归实现组合型枚举

例题中用递归实现了全排列，这里用递归实现组合数

组合数不要求顺序，但我们指定一个规则，要让程序按此规则枚举，否则会出现重复方案，我们采取当前位置的值比前一位大的方案，并且按照从小到大的顺序枚举，所以每次都是从某一个值到n。DFS的参数中除了当前的位数，还要加入一个起始值，表示从这个值开始到n为可用数字。

带分数

这是一道利用递归进行暴力枚举的题。但是可以在暴力基础上进行优化，可以成倍提高效率。

纯暴力：我们可以把答案的式子写成$$n=a+b/c$$​的形式，我们只需暴力枚举a,b,c的每一位的值，再转化为对应数值，判断正确后即为一种方案。

优化：我们将上述等式转化为$$nc=ac+b$$此时b可以通过计算得出，再将其拆成每一位，判断是否符合题意即可。这样我们就可以只枚举a和b。

费解的开关

此题为递推

我们要从每次操作中总结出规律。首先，当按开关的顺序是无所谓的。其次，当按两次同一个开关是无效的。所以我们从第一行开始看，要想使熄灭的灯亮起，就需要其对应的下一行的灯开关触发，每一行都如此，也就是说当初始状态给出后，每一行的操作都由上一行唯一确定。当到达最后一行后，还有未亮的灯，即为无解方案。
